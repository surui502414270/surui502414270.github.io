软件测试读书笔记

1.2 软件缺陷是什么

至少满足下列5个规则之一才称发生了一个软件缺陷

  1. 软件未实现产品规格说明书要求的功能
  2. 软件出现了产品规格说明书指明不应该出现的错误
  3. 软件实现了产品规格说明书未提到的功能
  4. 软件未实现产品规格说明书虽未明确提及但应该实现的目标
  5. 软件难以理解、不易使用、运行缓慢、或者（从测试员的角度看）最终用户会认为不好

1.5 软件测试员究竟做些什么

软件测试员的目标：尽可能早地找出缺陷，并确保其得以修复

2.3 软件开发生命周期模式

螺旋模式：一开始不必详细定义所有的细节，先定义重要功能，努力实现这些功能，接受客户反馈，然后进入下一阶段

  1. 确定目标、可选方案和限制条件
  2. 明确并化解风险
  3. 评估可选方案
  4. 当前阶段开发和测试
  5. 计划下一阶段
  6. 确定进入下一阶段的方法

软件测试员喜欢这个模式，因为他们很早就参与开发过程，有机会尽早发现问题，为项目节省时间和金钱

4.1 黑盒、白盒、静态、动态测试

  1. 黑盒测试（功能测试、行为测试）：只需知道软件要做什么，无法知道软件是如何运行的
  2. 白盒测试：可以访问程序的代码，并通过检查代码来协助测试
  3. 静态测试：只是检查和审核
  4. 动态测试：使用和运行软件

5.1 动态黑盒测试：带上眼罩测试软件

  1. 输入数据->接受输出->检验结果
  2. 动态黑盒测试又称行为测试，因为测试的是软件在使用过程中的实际行为
  3. 测试用例：进行测试时使用的特定输入，以及测试软件的过程步骤
  4. 等价类划分是指分步骤地把海量测试用例缩减得很小，但过程同样有效
  5. 数据测试等价类划分原则：边界条件；次边界条件；默认、空白、空值和无；非法、错误、不正确和垃圾数据
  6. 状态转换图应该表示出：软件可能进入的每一种独立状态；从一种状态进入另一种状态的输入和条件。进入或者退出状态的设置条件及输出结果
  7. 通过性状态测试：检查软件、描绘状态、尝试各种合法可能性、确认状态及其转换正常
  8. 失效性状态测试：竞争条件和时序错乱、重复、压迫和重负

6.1 检查代码

  1. 静态白盒测试是在不执行软件的条件下有条理地仔细审查软件设计、体系结构和代码，从而找出软件缺陷的过程，有时也称为结构化分析
  2. 正式审查是进行静态白盒测试的过程，包括4个基本要素：确定问题；遵守规则；准备；编写报告
  3. 编码标准和规范：可靠性；可读性/维护性；移植性
  4. 通用代码审查清单：数据引用错误（导致缓冲区溢出，安全缺陷）；数据声明错误；计算错误；比较错误；控制流程错误；子程序参数错误；输入/输出错误；其他检查

7.1 动态白盒测试：带上X光眼镜测试软件

动态白盒测试：查看代码功能（做什么）和实现方式（怎么做）得到的信息来确定哪些需要测试、哪些不需要测试、如何展开测试。（软件测试员可以查看并使用代码的内部结构，从而设计和执行测试，了解软件的运作方式有助于选择合适的测试方法。）

  1. 单元测试/模块测试、集成测试、系统测试
  2. 递增测试两条途径：自底向上（测试驱动调用被测试模块）和自顶向下（测试桩）
  3. 数据覆盖：数据流、次边界、公式和等式、错误强制
  4. 代码覆盖：程序语句和代码行覆盖、分支覆盖、条件覆盖

8 配置测试

  1. 确定所需硬件类型
  2. 确定有哪些厂商的硬件、型号和驱动程序可用
  3. 确定可能的硬件特性、模式和选项
  4. 将确定后的硬件配置缩减为可控制的范围
  5. 明确与硬件配置有关的软件唯一特性
  6. 设计在每一种配置中执行的测试用例
  7. 在每种配置中执行测试
  8. 反复测试直到小组对结果满意为止

9 兼容性测试

软件兼容性测试是指检查软件之间是否能够正确地交互和共享信息
兼容性测试记住3点：

  1. 对兼容软件的所有可能选择进行等价划分
  2. 研究适用于测试软件的高级/低级标准和规范
  3. 测试软件程序之间的不同数据流动方式

11 易用性测试

优秀UI具备的7个要素：符合标准和规范；直观；一致；灵活；舒适；正确；实用

13 软件安全性测试

黑客动机：挑战/成名，好奇，借用，恶意破坏（丑化、破坏、拒绝服务），偷窃

14 网站测试

  1. 黑盒测试：文本（当作文档对待）；超级链接；图片；表单（用于输入和选择信息的文本框、列表框和其他域）；对象和其他各种简单功能
  2. 灰盒测试（介于白盒和黑盒之间）适合网页测试，HTML（超文本标记语言）
  3. 白盒测试，网站系统结构和编程知识：动态内容；数据库驱动的网页；用编程方法创建的网页；服务器性能和加载；安全性
  4. 配置和兼容性测试，易用性测试

15 自动测试和测试工具

  1. 重复执行测试的过程称为回归测试
  2. 工具和自动化的主要属性：速度；效率；准确度和精确度；节省资源；仿真和模拟；坚持不懈
  3. 软件测试自动化可以执行测试用例，查找软件缺陷，分析看到的信息，记录结果
  4. 自动化：宏录制和回放；可编程的宏；完全可编程的自动测试工具
  5. 随机测试：模拟用户可能的操作，测试猴子
