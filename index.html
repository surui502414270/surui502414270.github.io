pgsql常用命令

1.启动 PostgreSQL：

pg_ctl -D /usr/local/var/postgres -l /usr/local/var/postgres/server.log start

2.关闭 PostgreSQL：

pg_ctl -D /usr/local/var/postgres stop -s -m fast

3.创建一个 PostgreSQL 用户

createuser username -P

4.创建数据库

createdb dbname -O username -E UTF8 -e

5.连接数据库

psql -U username -d dbname -h 127.0.0.1

6.显示已创建的数据库：

\l

7.连接数据库

\c dbname

8.显示数据库表

\d

9.创建一个名为 test 的表

CREATE TABLE test(id int, text VARCHAR(50));

10.插入一条记录

INSERT INTO test(id, text) VALUES(1, ‘sdfsfsfsdfsdfdf’);

11.查询记录

SELECT * FROM test WHERE id = 1;

12.更新记录

UPDATE test SET text = ‘aaaaaaaaaaaaa’ WHERE id = 1;

13.删除指定的记录

DELETE FROM test WHERE id = 1;

14.删除表

DROP TABLE tablename;

15.删除数据库

DROP DATABASE dbname;

前端总结

目录结构

Vue

  • Vue生命周期

共分为八个阶段
beforeCreate（实例刚被创建）el 和 data 并未初始化
created（实例创建完成，属性已绑定，但DOM 还未生成，$el属性还不存在）
beforeMount(模板编译/载入前)完成了 el 和 data 初始化 。此时还没有生成 html到页面上去
mounted（模板编译/载入后）完成挂载，也就是模板中的 html 渲染到 html 页面中，此时一般可以做一些 ajax 操作
beforeUpdate（数据更新时调用）
updated（更新完成后的钩子）
beforeDestroy（实例销毁之前调用。在这一步，实例仍然完全可用）
destroyed（Vue 实例销毁后调用）

  • Vue路由

    抽取共用的部分，比如导航栏，做主路由。之后再分子路由，如在论坛页面，将导航栏和用户信息抽取出来做公共部分放在 bbs/index 下。子路由分为 home、newart、hotart等，实现点击切换子路由。
    动态路由：如 article/：id，帖子具体页面，根据url 的 id 不同，得到不同数据。

  • Vuex

    Vuex 是一个专为 Vue.js 应用程序开发的状态管理模式。它采用集中式存储管理应用的所有组件的状态，并以相应的规则保证状态以一种可预测的方式发生变化。

    Getter：Vuex 允许我们在 store 中定义“getter”（可以认为是 store 的计算属性）。就像计算属性一样，getter 的返回值会根据它的依赖被缓存起来，且只有当它的依赖值发生了改变才会被重新计算。
    Mutation：更改 Vuex 的 store 中的状态的唯一方法是提交 mutation。Vuex 中的 mutation 非常类似于事件：每个 mutation 都有一个字符串的 事件类型 (type) 和 一个 回调函数 (handler)。
    Action类似于 mutation，不同在于：Action 提交的是 mutation，而不是直接变更状态。Action 可以包含任意异步操作。

  • 基于 Token的身份验证–JWT
    JWT就是一个字符串，经过加密处理与校验处理的字符串，形式为：A.B.C
    A由JWT头部信息header加密得到
    B由JWT用到的身份验证信息json数据加密得到
    C由A和B加密得到，是校验部分
    A.B.C就是生成的 token 了
    token应用流程？
    初次登录：用户初次登录，输入用户名密码
    密码验证：服务器从数据库取出用户名和密码进行验证
    生成JWT：服务器端验证通过，根据从数据库返回的信息，以及预设规则，生成JWT
    返还JWT：服务器的HTTP RESPONSE中将JWT返还
    带JWT的请求：以后客户端发起请求，HTTP REQUEST HEADER中的Authorizatio字段都要有值，为JWT

  • 路由拦截
    第一套 :
    第一层 meta是否控制路由需要验证
    第二层 store中token是否存在
    第三层 localstore中token是否存在，存在则进行token登录，成功将获得新token 存store
    第二套 :
    第一层 meta是否控制路由需要验证
    第二层 localstore中token是否存在

2.中间件

当我们谈论Iris中的中间件时，我们正在讨论在HTTP请求生命周期中的主要处理程序代码之前和/或之后运行代码。例如，日志记录中间件可能会将传入的请求细节写入日志，然后调用处理程序代码，然后编写有关日志响应的详细信息。关于中间件的一件很酷的事情是这些单元非常灵活且可重用。中间件只是一个Handler形式func(ctx iris.Context)，中间件在以前的中间件调用时正在执行ctx.Next()，这可以用于身份验证，即：如果登录，则会ctx.Next()引发错误响应。
